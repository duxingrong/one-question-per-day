# 看到哪些关键词，要想到滑动窗口？

## 1. “连续子数组 / 连续子串 / 子数组 / 子串”

比如：

```
最长连续子数组
最长子串
最短连续子数组
连续区间
```

滑动窗口解决的通常都是**连续的一段**。

注意区别：

```
子数组 / 子串：连续子序列：不一定连续
```

如果题目说的是“子序列”，一般就不是普通滑动窗口，可能是动态规划、贪心、双指针。


## 2. "最长/最短"

比如：

```
最长不重复子串
最长可替换子串
最短满足条件子数组
```

滑动窗口经常用于：

```
找最长合法窗口
找最短满足条件窗口
```

## 3. "最多k次/至多k个/不超过k"

比如：

```
最多修改k个元素
最多替换k个字符
最多包含k个不同字符
最多删除k个数
```

这种题目很典型：

```
窗口是否合法 = 某个代价 < = k
```

## 4. “满足某种条件的窗口”

比如：

```
不含重复字符
和大于等于 target
窗口内最多 k 个不同字符
窗口内 0 的个数最多为 k
```

只要题目能翻译成：

```
维护一个连续窗口
判断窗口是否合法
窗口不合法/合法时移动 left
```

就很可能是滑动窗口


# 滑动窗口的本质是什么？

滑动窗口本质上是在维护:

```
nums[left:right+1]

或者说：

s[left:right+1]
```
也就是一个连续区间。

他的核心是：

```
right 负责扩大窗口
left 负责缩小窗口
中间用某些变量维护窗口状态
ans 记录答案
```

# 思考流程

1. 窗口时什么？
2. 窗口里要维护的信息是什么？
3. 窗口什么时候合法？
4. 什么时候移动left? 求最长：窗口不合法时收缩 求最短：窗口合法时收缩
5. 什么时候更新答案？


# 怎么判断能不能用滑动窗口？

这点很重要。不是所有"连续子数组"都能用滑动窗口

滑动窗口通常需要满足一个性质：

```
当right向右扩展时，窗口状态可以被维护；
当窗口不满足条件时，可以通过移动left让他重新满足。

更直白的说：
窗口的变化是可控的
```

## 适合滑动窗口的情况

1. 全是正数的数组和问题
2. 字符频率类问题
3. 允许最多k次操作的问题

## 不适合普通滑动窗口的情况

如果数组里有负数，很多"和"的问题就不能直接用滑动窗口了

因为窗口右移时，和不一定增加；左移时，和也不一定减少。
这种情况下滑动窗口的"方向感"会失效



# 疑问解答

你担心的是：当right到最后一位时，left还没有继续往后走，那不是还有很多窗口没看吗？

确实没有枚举这些更短窗口，因为没必要，因为对于同一个right来说,如果当前的窗口已经合法，那么继续把left往右移动，只会让窗口变得更短，这些窗口的长度一定不会超过当前窗口。
而我们要的求的是：最长长度，所以没必要继续缩。

你可以记住这个原则：
```
求最长窗口：只在不合法时移动left,因为，合法时，窗口越长越好，不要主动缩小
求最短窗口：才会在窗口合法时继续移动left，我们想看看能布恩那个更短
```

滑动窗口不是为了遍历所有窗口，而是为了只保留“对答案有可能有贡献的窗口
